# Commit Guide

Follow conventional commits format.

- MUST use format: `<type>[scope]: <description>`
- MUST use lowercase
- MUST use imperative mood ("add" not "added")
- MUST keep under 50 characters
- NEVER add period at end
- NEVER add Claude Code attribution lines
- NEVER add Co-Authored-By lines
- Types: `feat`, `fix`, `docs`, `refactor`, `chore`, `style`, `perf`, `test`
- Example: `feat(ui): add ghost button`
